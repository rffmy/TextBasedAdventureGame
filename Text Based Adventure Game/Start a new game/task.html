Description

In this stage, you will implement the new game function and the game stats.

You need a game function that allows you to store all the necessary items and use them in your playthrough. The game stats will be saved once the player advances to another level.

You need to store important variables when creating a new game such as:

  • The username. You will need it in a later stage for the save file path.
  • The character attributes: Name, Species, and the Gender.
  • The inventory: Snack, Weapon, and Tool.
  • The difficulty level: 1- Easy, 2- Medium and 3- Hard. The user needs to select the difficulty level either by the word or the index.
  • The remaining lives: 5 for an easy difficulty level, 3 for a medium one, and 1 for a hard one. You don't need to take the number of lives as input, it should be calculated on the basis of the level that the user chooses.

You should get the game variables in the same order as in the Examples. If a player makes a mistake by choosing to create a new game, give them an option to go back to the main menu before creating a new player: Enter a user name to save your progress or type '/b' to go back.

The Level starts with 1 (the first level). The game is going to have 2 levels in total at the end.

Don't forget to handle incorrect input as before!

Objectives

In this stage, your program should:

  1. Create and store the game state with variables such as the inventory, character attributes, difficulty, remaining lives, and the username.
  2. Have the option to go back to the menu before creating a new game. Don't forget that it should handle all the letter cases.
  3. Before the new game starts, output the message Good luck on your journey! with the game stats below such as the character traits, inventory, difficulty, and the number of lives. Take a look at the Examples below to find out how you need to format the output.
