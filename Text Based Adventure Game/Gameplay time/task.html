Description

In this stage, you are going to implement the core game features and only the first level of the game.

You are not supposed to save the game yet, that will come later.

First, you need to read the contents of the story.json file. Download the file from the link and place it in a new subfolder story in your project directory. In this JSON file, you can see all of the items related to the story. In the story key, you will find each of the necessary scenes for the first level. Each level has 3 scenes.

When opening the JSON file for reading, make sure to use os.path.join() to build the path to where you've saved the file.

In the choices key, you will find the different choices of actions that the player can take. Also, you will see some of the words are wrapped in curly brackets like {weapon}. This means that you need to output the player's choice of weapon in these sentences. Output the right object in place of the bracketed word. Inspect or edit the JSON file to see or change what keywords are used as placeholders.

In the outcomes key, you will find the outcomes for each of the choices available to the player (i.e. outcome1 from lvl1, scene1 is the outcome of choice1 for lvl1). Each outcome string contains an expression in parentheses, for example: (life+1). These indicate what effect each outcome has on the game's flow and the variables tracking the game's state (like the number of lives remaining). Your program needs to take them out of the sentences before you print the outcome since they are not a part of the story.

Below is the table of the possible outcome effects:

life +1 increases the number of lives
life -1 decreases the number of lives
inventory + 'item_name' adds a new item to inventory
inventory - 'item_name' removes an item from inventory
move moves to the next segment of the story
repeat repeat the choices that were made in this segment of the story
save indicates the end of a level, saves the game
game_won a player has beaten the game, exits the program

These outcomes can be combined together to make two actions at the same time. For example: (life+1 and move)

Keep in mind that some of the outcomes, such as save and game_won, are going to appear in later stages. So, don't worry about them now.

In addition, you will see an outcome that is separated into different options: 

option1: You tried the key on the lock and the door opened.(inventory-'key' and move)

option2: You don't have a key to open the lock. (repeat)

This means that an action made by the player has two outcomes depending on the player's previous actions. In this instance, for the option1 to happen, the player should have a key in the inventory. However, if the player doesn't have a key, the option2 happens.

Once you have the story, you can move to the core game loop. The game loop should check for the number of lives and if there are no lives remaining, the game is over. Also, check for the level of the game. If the player reaches the second level, your game should show it. However, since you are only implementing the first level in this stage, just output the Level 2 message and end the program.

Don't forget to print the Goodbye! message every time your program ends

Once the core game loop is set up, it's time to implement the gameplay features. Introduce your story to a player, then give a player some options to continue the game. The options should be numbered so that the player can type the number to choose the option. Limit the options to three, just to keep it simple and show the outcome.

If your character dies, the game should start from the beginning of the level, decrease the number of the remaining lives but preserve the content of the inventory.

You also need to implement the following commands: /i, /c, /h:

/i shows the content of the inventory
/c shows the character traits and the remaining lives
/h the help page; shows the commands available to a player

The player should be able to exit the game by typing /q anytime. Don't forget to ask for the confirmation. Don't forget to show a message when an unknown input occurs.

You can always write your own story for this project. In fact, it is encouraged to do so. However, make sure that you use the same character traits and inventory items in your story. Also, make sure that in your story, the player always has the weapon, the tool and the snack.

Objectives

In this stage, your program should:

  1. Read the story folder content.
  2. Create a core game loop.
  3. Take inputs from the player for different actions.
  4. Increase the number of lives.
  5. Decrease the number of lives.
  6. Show the player inventory.
  7. Add an item to the inventory.
  8. Remove the item from the inventory.
  9. Show the character traits, including the remaining lives.
  10. Show the help page.
