Description

The next thing we are going to implement is the saving feature.

We need to save the game progress after the end of every level. By saying "the game progress", we mean the game stats introduced in the previous stage.

In your project directory, make a folder called game with a subfolder named saves. For now, it is empty, but to implement your save functionality, your program will create new JSON files and write the items from the game stats. Each JSON save file will be named with the player's username, such that its filepath looks like: game/saves/{username}.json. This way, each user will have their own save file.

Write the following game stats to your save file: the character traits, in-game inventory, difficulty, remaining lives, and the current level. To understand how to write the stats to file you can check this example save JSON file for the user called 'new_user': game/saves/new_user.txt. Follow the same structure exactly as you see in this example text file, with the necessary keys and values in the JSON.

As mentioned before, if you are using your own story, make sure you always include the weapon, the tool and the snack in the inventory and the game stats.

Don't forget to change the level. When a player advances to the next level, increase the current level count by 1 and save the game.

You can add some color to your text if you want to make it more fun. The Termcolor library can help you with that. Our tests do not check the color of your text.

Objectives

In this stage, your program should:

  1. Create save files for each user.
  2. Write the game state when the player advances to another level. You can also add some color to your text, but we won't check it.
Example

The greater-than symbol followed by a space (> ) represents the user input. Note that it's not part of the input.
